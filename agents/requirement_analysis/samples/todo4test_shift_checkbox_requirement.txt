【需求名称】TODO4Test：Shift 键连续勾选行选择标记（邮箱列表 Demo）

【业务背景】
页面模拟邮件客户端的收件箱列表：多行内容，每行左侧有可在「未选 / 已选」之间切换的标记。当前仓库为单页静态 HTML（index-START.html），样式与结构已具备，但「交互逻辑」尚未实现（<script> 为空）。

【目标】
在保持现有 HTML 结构与 CSS 的前提下，仅使用宿主语言脚本与 DOM API（不引入第三方 UI 框架或库），在页面中实现以下行为：

1. 用户可像平常一样单独点击某行的选择标记进行选中/取消。
2. 当用户按住 Shift 键并点击另一行的标记时：从「上一次被点击的行」到「当前被点击的行」（含两端）区间内，全部行应设为「已选」。
3. 若本次点击时未按住 Shift，则仅切换当前行的选中状态，并更新「最近一次点击锚点」，供下一次配合 Shift 使用。
4. 行为需贴近常见桌面邮件客户端的列表多选体验；脚本放在入口 HTML 的 <script> 内或由方案拆出独立脚本；不得破坏现有 class 语义（如 .inbox、.item）。

【非目标】
- 不新增远端存储、不向服务端上报选中集合。
- 不改变整体视觉风格与布局（除极小可用性补丁外）。

【验收标准（可测试）】
- 逐行单击，选中状态切换正确。
- 先点行 A，再按住 Shift 点行 B：A 与 B 之间（含）全部变为已选。
- 多次 Shift 范围操作后锚点更新正确；浏览器控制台不应出现未捕获异常。

【技术约束】
- 技术栈：静态 HTML + 内联或外链原生脚本；仓库可无包管理清单，后续流水线侧自行补齐运行环境。
- 基线入口：index-START.html；交付入口文件名由后续方案在设计变更清单中约定。

【验收环境】
- 现代桌面浏览器（Chromium 系为主）即可。
