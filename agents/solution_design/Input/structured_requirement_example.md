## 2. 背景与问题说明

### 2.1 背景

当前用户在登录失败后只能看到通用错误提示，无法区分密码错误、账号不存在或网络异常。

### 2.2 当前问题

- 用户无法判断失败原因。
- 重试时缺少明确反馈。
- 网络异常和账号错误的提示不区分。

### 2.3 目标用户

- 主要用户：未登录用户
- 使用场景：登录账号
- 触发入口：登录页面提交按钮

---

## 3. 需求目标

### 3.1 本次要实现什么

1. 根据不同失败原因展示不同错误文案。
2. 请求 pending 时禁用提交按钮并显示 loading 状态。
3. 网络异常时允许用户重试。

### 3.2 本次不做什么

1. 不修改登录接口协议。
2. 不新增第三方 UI 组件库。
3. 不重构全局认证流程。

---

## 6. 验收标准

- [ ] 密码错误时展示明确提示。
- [ ] 账号不存在时展示明确提示。
- [ ] 网络异常时展示可重试反馈。
- [ ] Loading 期间提交按钮不可重复点击。
- [ ] 不影响注册、忘记密码等无关流程。
