## 附录：v1 冻结摘要（全选 A）

- **时机**：测试后仍可出报告；测试失败时结论带风险说明（`risk_note`）。  
- **决策**：Agent 仅建议；合入一律人工；不强制 CI 失败（可自行接）。  
- **输入**：`code_changes.diff` + 单一设计 MD + 必选 `code_test_result.json`，可选 log / `requirement_spec` / `codegen_result.json`（不一致仅 `warnings`）。  
- **输出**：`review_result.json` + 可选 `review_report.md` + `feedback_review.md`（与 `--repair-feedback` 兼容）；`severity` 五档；无 `risk_score`；含 `test_gaps[]`。  
- **审查范围**：前端为主 + 其它语言粗提示；正确性对齐方案+diff+可选需求摘录。  
- **安全/规范**：无强制外链扫描；脱敏默认；嵌入式 policy + 可选 `--policy-pack`。  
- **运行**：与 CodeGen/CodeTest 同 env；本机或 CI；`max_llm_calls` + 截断；报告含 schema 版本、低温度。  
- **工程**：`CodeReview/` 独立包与 CLI；不与 `delivery_integration` 强绑。  
- **验收**：见 §8（8.1–8.3 已填 **需要**）。
